GUI redesign — concept mockups

Cleaner, slicker Demeter

Three directions for the same screen. They’re static HTML — open each and click around the collapsibles. None of this is wired to the backend; it’s to react to, not to ship as-is.

What makes today’s screen feel busy

  • Red everywhere. The whole value box fills red for “differs” and “undefined”, so a normal card looks alarming. There’s no calm “matches” or “not managed” state.
  • The header is a control dump. Import/Export, Show-offline, Auto-reboot, Scan-every and account info all sit in one undifferentiated row.
  • “Collapse All” appears three times (page, frame, slot) — repetitive and ambiguous.
  • The three value columns aren’t labelled. Coloured box vs. input — which is live, which is the target? No headers explain it.
  • Weak hierarchy. Frame, slot, group and row are all the same dark grey; the nesting is hard to read at a glance.
  • Wasted width + low density. Octet inputs sprawl across the screen while tall rows show only a handful of settings.
  • Label bug. “F101- -” — number/name/dashes concatenated awkwardly.

What all three mockups change

  • Status as a calm signal: a dot + “On device → Target”. Green = matches, amber = will change, grey = not managed. Red is reserved for genuinely unreachable.
  • Labelled columns so the live value vs. the target is unambiguous.
  • Real hierarchy: frame = elevated card with a status strip + proper title (“F101 · Some frame”), slot = nested panel, groups = collapsible with a per-group change count.
  • A decluttered top bar: global toggles fold into a ⚙ menu; the version/account sit quietly on the right.
  • A “changes pending” summary so you always know what a blast will actually do.
